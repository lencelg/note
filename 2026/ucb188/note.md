# Agents

在人工智能中，核心问题是创造**理性智能体**，即一个具有目标或偏好，并试图根据这些目标执行一系列能产生最佳/最优期望结果的行动的实体。理性智能体存在于**环境**中，该环境取决于智能体的具体实例化。智能体使用**传感器**与环境交互，并使用**执行器**对环境施加作用。一个非常简单的例子：跳棋智能体的环境是它与对手对弈的虚拟棋盘，棋子的移动就是动作。环境及其中的智能体共同构成了一个**世界**。

**反应式智能体**是不考虑其行动后果，仅根据当前世界状态选择行动的智能体。这类智能体的性能通常不如**规划型智能体**，后者维护一个世界模型，并使用该模型模拟执行各种动作。然后，智能体可以确定行动的假设后果，并选择最佳的一个。这就是模拟的“智能”，因为它正是人类在试图确定任何情况下最佳可能走法时所做的事情——提前思考。

为了定义任务环境，我们使用 **PEAS**（性能度量、环境、执行器、传感器）描述。**性能度量**描述了智能体试图提高什么效用。**环境**概括了智能体在哪里行动以及什么影响智能体。**执行器**和**传感器**是智能体作用于环境并从中接收信息的方法。

智能体的设计在很大程度上取决于其行动的环境类型。我们可以通过以下方式对环境的类型进行刻画。

在**部分可观测**环境中，智能体没有关于状态的完整信息，因此智能体必须对世界状态有一个内部的估计。这与**完全可观测**环境形成对比，在完全可观测环境中，智能体拥有关于其状态的完整信息。**随机性**环境在转移模型中存在不确定性，即在特定状态下执行一个动作可能以不同的概率产生多种可能的结果。这与**确定性**环境形成对比，在确定性环境中，在状态下执行一个动作有且只有一个保证发生的结果。在**多智能体**环境中，智能体与其他智能体一起在环境中行动。出于这个原因，

* 智能体可能需要随机化其行动，以避免被其他智能体“预测”。
* 如果环境在智能体行动时不发生变化，那么这种环境被称为**静态**的。这与**动态**环境形成对比，动态环境会随着智能体的交互而改变。
* 如果环境具有已知的物理规律，那么转移模型（即使是随机的）对智能体来说是已知的，它可以在规划路径时使用该模型。如果物理规律未知，智能体需要有意识地采取行动来学习未知的动态特性。
## Search Problem
### state space

为了创建一个理性的规划智能体，我们需要一种方法来数学地表达智能体将要存在的给定环境。为此，我们必须形式化地表达一个**搜索问题**——给定我们智能体的当前状态（它在环境中的配置），我们如何才能以最佳方式达到满足其目标的新状态？一个搜索问题由以下要素组成：

*   **状态空间** - 在给定世界中所有可能状态的集合。
*   **一组可用动作** - 在每个状态下可用的动作。
*   **转移模型** - 描述在当前状态下执行特定动作后到达的下一个状态。
*   **动作代价** - 执行一个动作从一个状态转移到另一个状态时产生的成本。
*   **起始状态** - 智能体最初所处的状态。
*   **目标测试** - 一个函数，将状态作为输入，并判断它是否为目标状态。

## Uniform search

找到从起始状态到目标状态的规划的标准协议是维护一个由从搜索树派生的部分规划组成的**外部边界**。我们通过从边界中移除一个节点（根据我们给定的策略选择）来不断**扩展**我们的边界，该节点对应一个部分规划，并用其所有子节点替换它到边界上。从边界中移除一个元素并用其子节点替换，对应于丢弃一个长度为 $n$ 的规划，并考虑所有由它产生的长度为 $(n + 1)$ 的规划。我们继续这个过程，直到最终从边界中移除一个目标状态，此时我们得出结论：对应于被移除目标状态的部分规划实际上是从起始状态到目标状态的一条路径。

### Depth-First Search
![](./img/DFS)

*   **description** - 深度优先搜索（DFS）是一种探索策略，总是选择从起始节点开始最深的边界节点进行扩展。
*   **fringe representation** - 移除最深的节点并用其子节点替换到边界上，必然意味着子节点现在是新的最深节点——它们的深度比前一个最深节点的深度大一。**后进先出（LIFO）stack** could the represent fringe of DFS
*   **Complete** - not complete if there is a cycle
*   **Optimality** - 深度优先搜索只是在搜索树中找到“最左边”的解，不考虑路径成本，因此不是最优的。
*   **Time complexity** - 在最坏情况下，深度优先搜索可能最终探索整个搜索树。因此，对于最大深度为 $m$ 的树，DFS 的运行时间为 $O(b^m)$。
*   **Space complexity** - 在最坏情况下，DFS 在边界上维护 $m$ 个深度级别上各 $b$ 个节点。这是一个简单的推论：一旦某个父节点的 $b$ 个子节点入队，DFS 的特性允许在任何给定时间点只探索这些子节点中某一个的子树。因此，DFS 的空间复杂度为 $O(bm)$。
### Breadth-First Search
![](./img/BFS)

*   **descirption** - 广度优先搜索是一种探索策略，总是选择从起始节点开始最浅的边界节点进行扩展。
*   **fringe representation** - 如果我们想在更深的节点之前访问更浅的节点，我们必须按照节点插入的顺序访问它们。因此，我们需要一个输出最早入队对象的结构来表示我们的边界。为此，BFS 使用**先进先出（FIFO）队列**，它正是做这个的。
*   **Comlete** - BFS is complete 
*   **Optimal** - BFS usually not optimal, it treats `each node has the same cost`, so it is `special case of UCS`
*   **Time complexity** - 在最坏情况下，我们必须搜索 $1 + b + b^{2} + \ldots + b^{s}$ 个节点，因为我们遍历了从 1 到 $s$ 的每一深度的所有节点。因此，时间复杂度为 $O(b^{s})$。
*   **Space complexity** - 在最坏情况下，边界包含最浅解所在层级的所有节点。由于最浅解位于深度 $s$，此深度有 $O(b^{s})$ 个节点。
###  Uninform Cost Search
![](./img/UCS)

*   **description** - 一致代价搜索（UCS），我们的最后一个策略，是一种探索策略，总是选择从起始节点开始成本最低的边界节点进行扩展。
*   **fringe representation** - 为了表示 UCS 的边界，通常选择基于堆的**priority queue**，其中给定入队节点 $v$ 的优先级是从起始节点到 $v$ 的路径成本，即 $v$ 的**后向成本**。直观地说，当我们移除当前最小成本路径并用其子节点替换时，以这种方式构建的优先队列会自我重新调整，以保持按路径成本所需的排序。
*   **Complete** - UCS is complete
*   **Optimality** - if all the cost is non-negative, UCS is Optimal, it has the same strategy as `Dijkstra algorithm`, `but if has negative cost, it is not optimal`
*   **Time complexity** - 让我们将最优路径成本定义为 $C^{*}$，状态空间图中两个节点之间的最小成本定义为 $\epsilon$。那么，我们大致需要探索深度从 1 到 $C^{*} / \epsilon$ 的所有节点，导致运行时间为 $O(b^{C^{*} / \epsilon})$。
*   **Space complexity** - 大致上，边界将包含最便宜解所在层级的所有节点，因此 UCS 的空间复杂度估计为 $O(b^{C^{*} / \epsilon})$。

### Greedy Search 
![](./img/greedy%20search)

*   **description** - always expand to node that seems cloest
*   **fringe representation** - 贪婪搜索与 UCS 的操作方式相同，都使用优先队列作为边界表示。区别在于，UCS 使用计算得到的后向成本（到该状态的路径上的边权重之和）来分配优先级，而贪婪搜索则使用启发式值形式的估计前向成本。
*   **complete and optimal** - 如果存在目标状态，贪婪搜索不能保证找到它，也不能保证最优性，尤其是在选择了非常差的启发式函数的情况下。它在不同场景下的表现通常相当不可预测，可能从直接走向目标状态，到像引导不良的 DFS 一样探索所有错误区域。

### A* Search

*   **description** - A* 搜索是一种探索策略，总是选择具有最低估计总成本的边界节点进行扩展，其中总成本是从起始节点到目标节点的全部成本。
*   **fringe representation** - 与贪婪搜索和 UCS 一样，A* 搜索也使用优先队列来表示其边界。同样，唯一的区别在于优先级的选择方法。A* 将 UCS 使用的总后向成本（到该状态的路径上的边权重之和）与贪婪搜索使用的估计前向成本（启发式值）相加，有效地给出了从起始到目标的估计总成本。鉴于我们希望最小化从起始到目标的总成本，这是一个极佳的选择。
*   **complete and optimality** - `given suitable heuristics，A* is complete and optimal`,它结合了我们目前所学的所有其他搜索策略的优点，将贪婪搜索通常的高速度与 UCS 的最优性和完备性融为一体！

### Admissibility and Consistency

现在我们已经讨论了启发式函数以及它们在贪婪搜索和 A* 搜索中的应用，让我们花些时间讨论一下什么构成了一个好的启发式函数。为此，我们首先用以下定义重新表述 UCS、贪婪搜索和 A* 中用于确定优先队列排序的方法：

*   $g(n)$ - 表示 UCS 计算的总后向成本的函数。
*   $h(n)$ - 启发式函数值，或估计前向成本，被贪婪搜索使用。
*   $f(n)$ - 表示估计总成本的函数，被 A* 搜索使用。$f(n) = g(n) + h(n)$。

在探讨什么构成“好”的启发式函数这个问题之前，我们必须首先回答：无论我们使用何种启发式函数，A* 是否能保持其完备性和最优性？事实上，很容易找到会破坏这两个宝贵属性的启发式函数。例如，考虑启发式函数 $h(n) = 1 - g(n)$。无论搜索问题是什么，使用这个启发式函数都会得到：

$f(n) = g(n) + h(n)= g(n) + (1 - g(n))= 1$

因此，这样的启发式函数将 A* 搜索简化为 BFS，且所有边成本相等。正如我们已经展示的，在边权重不一般情况下，BFS 不能保证最优性。

在使用 A* 树搜索时，最优性所需的条件被称为**可采纳性**。可采纳性约束指出，可采纳启发式函数估计的值既不是负数也不会高估。将 $h^{*}(n)$ 定义为从给定节点 $n$ 到达目标状态的真实最优前向成本，我们可以将可采纳性约束用数学公式表示如下：

$\forall n,0\leq h(n)\leq h^{*}(n)$

1. $g(A) < g(B)$。由于已知 $A$ 是最优的，而 $B$ 是次优的，我们可以得出结论，$A$ 到起始状态的后向成本低于 $B$。

2. $h(A) = h(B) = 0$，因为我们已知启发式函数满足可采纳性约束。由于 $A$ 和 $B$ 都是目标状态，从 $A$ 或 $B$ 到目标状态的真实最优成本就是 $h^{*}(n) = 0$；因此 $0 \leq h(n) \leq 0$。

3. $f(n) \leq f(A)$，因为通过 $h$ 的可采纳性，$f(n) = g(n) + h(n) \leq g(n) + h^{*}(n) = g(A) = f(A)$。经过节点 $n$ 的总成本最多等于 $A$ 的真实后向成本，也就是 $A$ 的总成本。

我们可以结合陈述 1 和 2 得出 $f(A) < f(B)$，如下所示：

$f(A) = g(A) + h(A) = g(A) < g(B) = g(B) + h(B) = f(B)$

将上述推导出的不等式与陈述 3 结合，一个简单的结果是：

$f(n)\leq f(A)\wedge f(A)< f(B)\Longrightarrow f(n)< f(B)$

因此，我们可以得出结论，$n$ 在 $B$ 之前被扩展。因为我们已经对任意的 $n$ 证明了这一点，所以我们可以得出结论，$A$ 的所有祖先节点（包括 $A$ 自身）都在 $B$ 之前被扩展

我们之前发现的树搜索的一个问题是，在某些情况下它可能永远找不到解，陷入在状态空间图中无限循环搜索同一个环。即使我们的搜索技术不涉及这种无限循环，我们也常常会多次访问同一个节点，因为存在多种方式到达同一个节点。这会导致指数级增长的工作量，自然的解决方案是简单地跟踪你已经扩展过的状态，并且永远不再扩展它们。更明确地说，在利用你选择的搜索方法时，维护一个已扩展节点的“已到达”集合。然后，确保在扩展之前每个节点都不在集合中，如果不在，则在扩展后将其添加到集合中。加入这种优化的树搜索被称为**graph search**<sup>1</sup>，其伪代码如下所示：

注意，在实现中，将已到达集合存储为`不相交集`而非列表至关重要。将其存储为列表需要 $O(n)$ 的成员检查操作，这会消除图搜索旨在提供的性能改进。图搜索还有一个额外的警告：即使使用可采纳的启发式函数，它也往往会破坏 A* 的最优性

$\forall A,C\quad h(A) - h(C)\leq cost(A,C)$

**定理**。对于给定的搜索问题，如果启发式函数 $h$ 满足一致性约束，则在该搜索问题上使用带有 $h$ 的 A* 图搜索将产生最优解。

**证明**。为了证明上述定理，我们首先证明，当运行具有一致启发式的 A* 图搜索时，每当我们移除一个节点进行扩展时，我们就已经找到了到达该节点的最优路径。

使用一致性约束，我们可以证明沿着任何规划路径上的节点的 $f(n)$ 值是非递减的。定义两个节点 $n$ 和 $n^{\prime}$，其中 $n^{\prime}$ 是 $n$ 的子节点。那么：

$f(n^{\prime}) = g(n^{\prime}) + h(n^{\prime}) = g(n) + cost(n, n^{\prime}) + h(n^{\prime}) \geq g(n) + h(n) = f(n)$

如果对于一条路径上的每个父子对 $(n,n^{\prime})$，都有 $f(n^{\prime}) \geq f(n)$，那么沿着该路径 $f(n)$ 的值必定是非递减的。我们可以检查上面的图在 $f(A)$ 和 $f(C)$ 之间是否违反了此规则。有了这个信息，我们现在可以证明，每当一个节点 $n$ 被移出进行扩展时，它的最优路径已经被找到。用反证法假设这是错误的——即当 $n$ 从边界中移除时，找到的到 $n$ 的路径是次优的。这意味着在边界上必定存在 $n$ 的某个祖先节点 $n^{\prime \prime}$，它从未被扩展，但位于到 $n$ 的最优路径上。矛盾！我们已经证明沿着路径的 $f$ 值是非递减的，因此 $n^{\prime \prime}$ 应该在 $n$ 之前被移出扩展。

要完成证明，我们只需证明最优目标 $A$ 将始终在任何一个次优目标 $B$ 之前被移出扩展并返回。这是显而易见的，因为 $h(A) = h(B) = 0$，所以

$[f(A) = g(A) < g(B) = f(B)$]

正如我们在可采纳性约束下证明 A* 树搜索最优性时一样。因此，我们可以得出结论，在一致启发式下，A* 图搜索是最优的。

在继续之前，从以上讨论中提炼出几个重点：对于有效（可采纳/一致）的启发式函数，根据定义，对于任何目标状态 $G$，必须满足 $h(G) = 0$。此外，一致性不仅仅是比可采纳性更强的约束，一致性蕴涵可采纳性。这源于一个简单的事实：如果没有边成本被高估（如一致性所保证的），那么从任何节点到目标的总估计成本也不会被高估。

### Dominance

现在我们已经确立了可采纳性和一致性的性质，以及它们在维持 A* 搜索最优性中的作用，我们可以回到最初的问题：如何创建“好”的启发式函数，以及如何判断一个启发式函数比另一个更好。这方面的标准度量是**主导性**。如果启发式函数 $a$ 主导启发式函数 $b$，那么对于状态空间图中的每一个节点，$a$ 估计的目标距离都大于 $b$ 估计的目标距离。数学上表示为：

$\forall n : h_a(n) \geq h_b(n)$

主导性非常直观地抓住了“一个启发式函数比另一个更好”的概念——如果一个可采纳/一致的启发式函数主导另一个，它必定更好，因为它总能更接近地估计从任何给定状态到目标的距离。此外，**平凡启发式**定义为 $h(n) = 0$，使用它会将 A* 搜索简化为 UCS。所有可采纳的启发式函数都主导平凡启发式。平凡启发式通常位于搜索问题的半格子的底部，这是一个主导性层级结构，它位于最底层。下面是一个半格子的示例，其中包含了从底部的平凡启发式到顶部的精确目标距离的各种启发式 $h_a$、$h_b$ 和 $h_c$：

作为一般规则，应用于多个可采纳启发式函数的 **max** 函数也总是可采纳的。这仅仅是因为对于任何给定状态，所有启发式函数输出的值都受到可采纳性条件 $0 \leq h(n) \leq h^{*}(n)$ 的约束。该范围内数字的最大值也必定落在同一范围内。对于多个一致的启发式函数，也可以轻松证明同样成立。对于任何给定的搜索问题，通常的做法是生成多个可采纳/一致的启发式函数，并计算它们输出值的最大值，以生成一个主导（因此优于）它们各自单独的启发式函数。

## Simulated Annealing
我们将介绍的第二个局部搜索算法是模拟退火。模拟退火旨在结合随机游走（随机移动到邻近状态）和爬山法，以获得一个完备且高效的搜索算法。在模拟退火中，我们允许移动到可能降低目标值的状态。

该算法在每个时间步选择一个随机移动。如果该移动导致更高的目标值，则总是被接受。如果导致更小的目标值，则以一定的概率接受该移动。这个概率由温度参数决定，温度参数初始值很高（允许更多“坏”的移动），并根据某种“调度”逐渐降低。理论上，如果温度降低得足够慢，模拟退火算法将以接近 1 的概率达到全局最大值。

## Local beam search

局部束搜索是爬山搜索算法的另一种变体。两者之间的主要区别在于，局部束搜索在每次迭代中跟踪 $k$ 个状态（线程）。该算法从随机初始化 $k$ 个状态开始，并在每次迭代中按照爬山法的方式获取 $k$ 个新状态。这些不仅仅是常规爬山算法的 $k$ 个副本。关键在于，该算法从所有线程的所有后继状态的完整列表中选出 $k$ 个最佳后继状态。如果任何一个线程找到了最优值，算法就会停止。

这 $k$ 个线程之间可以共享信息，使得“好”的线程（目标值高的线程）也能将该区域的其他线程“吸引”过来。

![](./img/beam%20search)

局部束搜索也像爬山法一样容易陷入“平坦”区域。类似于随机爬山法的`随机束搜索(Stochastic beam search)`可以缓解这个问题。

## Genetic algorithm

最后，我们介绍遗传算法，它是局部束搜索的一种变体，也广泛用于许多优化任务。顾名思义，遗传算法从进化论中汲取灵感。遗传算法开始时是束搜索，有 $k$ 个随机初始化的状态，称为种群。状态（称为个体）被表示为有限字母表上的字符串。

让我们回顾一下讲座中介绍的八皇后问题。作为回顾，八皇后是一个约束满足问题，我们希望将八个皇后放在一个 8x8 的棋盘上。约束满足的解将没有任何攻击对的皇后，即在同一行、同一列或同一对角线上的皇后。之前介绍的所有算法都是解决八皇后问题的可能方法。

图 4.3

<center>(a) 八皇后问题：在棋盘上放置八个皇后，使得没有皇后攻击另一个。（皇后攻击同一行、列或对角线上的任何棋子。）这个位置几乎是一个解，除了第四列和第七列的两个皇后沿着对角线相互攻击。(b) 一个启发式代价估计为 $h = 17$ 的八皇后状态。棋盘显示了通过在列内移动皇后得到的每个可能后继的 $h$ 值。有 8 个移动并列最佳，$h = 12$。爬山算法将选择其中一个。</center>

对于遗传算法，我们用 1 到 8 之间的数字表示每个皇后，代表每个皇后在其列中的位置（图 4.6 中的列 (a)）。每个个体使用评估函数（适应度函数）进行评估，并根据该函数的值进行排名。对于八皇后问题，这是非攻击（无冲突）的皇后对的数量。

<center>图 4.6 遗传算法，以表示八皇后状态的数字串为例。(a) 中的初始种群根据 (b) 中的适应度函数进行排名，得到 (c) 中用于交配的对。它们产生 (d) 中的后代，然后在 (e) 中进行突变。</center>

选择状态进行“繁殖”的概率与该状态的值成正比。我们通过从这些概率中抽样来选择要繁殖的状态对（图 4.6 中的列 (c)）。后代通过在交叉点对父代字符串进行交叉而产生。交叉点是为每对随机选择的。最后，每个后代以独立的概率易受某些随机突变的影响。遗传算法的伪代码如下所示。

```text
函数 遗传算法(种群, 适应度函数) 返回一个个体
    输入: 种群, 一组个体
         适应度函数, 用于衡量个体适应度的函数
    重复
        新种群 ← 空集
        i = 1 到 种群大小 执行
            x ← 随机选择(种群, 适应度函数)
            y ← 随机选择(种群, 适应度函数)
            子代 ← 繁殖(x, y)
            如果 (小的随机概率) 那么 子代 ← 突变(子代)
            将子代添加到新种群
        种群 ← 新种群
    直到 某个个体足够适应，或者足够的时间已经过去
    返回 种群中根据适应度函数的最佳个体

函数 繁殖(x, y) 返回一个个体
    输入: x, y, 父代个体
    n ← 长度(x); c ← 从 1 到 n 的随机数
    返回 连接(子串(x, 1, c), 子串(y, c+1, n))
```

与随机束搜索类似，遗传算法试图在探索状态空间的同时向上移动，并在线程之间交换信息。它们的主要优势在于使用交叉——已经进化并导致高评估值的大块字母可以与其他此类块结合，产生具有高总分的解。
